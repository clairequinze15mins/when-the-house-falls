# 📝 贡献指南（Contributing Guide）  
*房子塌了怎么办 / When the House Falls*

欢迎加入《房子塌了怎么办》开放协作项目！  
无论你擅长分析、叙事创作，还是提出新想法，都可以在这里贡献智慧。  
即使你没有使用 GitHub 的经验，也能按照本指南参与。

---

## 一、基础概念（新手必读）

| 术语 | 说明 |
|------|------|
| **Repository（仓库）** | 项目的“总文件夹”，包含所有文档和资源。 |
| **Fork（复刻）** | 把主项目复制到你的账号，便于自由修改。 |
| **Branch（分支）** | 在同一份内容里开出一个平行版本，便于实验或创作不同剧情。 |
| **Commit（提交）** | 保存一次修改，相当于存档。 |
| **Pull Request（PR）** | 请求把你的修改合并回主项目。 |
| **Issue（议题）** | 提出讨论、问题或创意建议。 |
| **Discussion（讨论区）** | 更开放的讨论场所，用于交流思路或发起讨论。 |

---

## 二、参与流程概览

1. 浏览已有内容  
2. 选择内容类型（分析 / 叙事 / 大纲）  
3. Fork 仓库  
4. 新建文件夹或修改内容  
5. 提交 Pull Request（PR）  
6. 参与讨论或提出 Issue  

---

## 三、步骤说明

### Step 1. 浏览项目
- 打开仓库首页，阅读 `README.md` 和 `outline/` 下的内容；  
- 查看已有的 `analyses/` 与 `storylines/` 文件夹；  
- 进入 [Discussions](../../discussions) 参与讨论。

---

### Step 2. 选择内容类型

| 类型 | 目录 | 内容示例 |
|------|------|----------|
| **分析类（Analysis）** | `/analyses/` | 战略推演、社会走向分析、风险评估、方法论应用等 |
| **叙事类（Storyline）** | `/storylines/` | 虚构故事、寓言、人物命运线、另类历史演绎等 |
| **设定类（Outline）** | `/outline/` | 世界观、关键问题、方法论说明 |

> 不确定归类，可先在 Discussions 征求意见。

---

### Step 3. Fork 仓库
1. 点击右上角 **Fork** 将项目复制到你的账号；  
2. 你现在有了自己的副本，可以自由修改和添加内容。

---

### Step 4. 新建分支（可选）
- 在你的 Fork 页面，点击左上角的分支选择菜单 → “New branch”；  
- 输入分支名（如 `alt-ending`），点击创建；  
- 在该分支中编辑内容，便于尝试不同方案或故事版本。  

---

### Step 5. 添加或修改内容
方式一：**网页直接编辑**  
1. 进入对应文件夹，例如 `/storylines/` 或 `/analyses/`；  
2. 点击 “Add file” → “Create new file”；  
3. 输入文件名（如 `storylines/zhang-utopia/summary.md`）；  
4. 编辑内容；  
5. 填写底部说明（例如“添加张伟的虚构故事分支”）；  
6. 点击 “Commit changes” 保存。

方式二：**本地编辑再上传**  
- 使用 GitHub Desktop 或其他 Git 工具，将 Fork 下载到本地编辑，再 push 回去。

---

### Step 6. 提交 Pull Request（PR）
1. 完成修改后，点击 “Compare & pull request”；  
2. 填写 PR 标题，建议用 `[SCENARIO]`、`[FICTION]` 或 `[ANALYSIS]` 分类；  
3. 描述你的主要修改、假设、目的或希望得到的反馈；  
4. 提交 PR，等待维护者审核或合并。

---

### Step 7. 参与讨论与反馈
- 使用 **Discussions** 发起或加入讨论；  
- 使用 **Issues** 提出明确修改建议或新议题；  
- 可以在 PR 下评论，给其他贡献者提供反馈。

---

## 四、命名与结构建议

### 文件夹命名示例

analyses/lihua-collapse/
storylines/zhangwei-utopia/
outline/methods/


### 文件内容建议
| 文件 | 作用 |
|------|------|
| `README.md` | 概述：背景、假设、方法、结论 |
| `content/` | 主体文本、图表或数据 |
| `references.md` | 参考资料、来源或灵感 |

---

## 五、行为准则（Code of Conduct）
1. 尊重他人观点，保持开放与礼貌；  
2. 不发布违法、煽动或敏感操作内容；  
3. 不泄露他人隐私；  
4. 接受维护者对违规内容的删除或警告；  
5. 鼓励理性分析与创意表达。

---

## 六、常见问题
- **不会操作？** → 在 Discussions 中发帖，标注 `[HELP]`；  
- **PR 没被合并？** → 维护者可能在审核或等待讨论，可在 PR 下留言说明意图；  
- **不确定放哪？** → 先在 Discussions 征求意见。
